## 1. Overall UX, navigation, and visual system

### 1.1 Site map (top navigation)

Top nav (desktop):

- Logo (top‑left)
- Home
- Platform
- AI Security Tools
- Consulting \& Training
- Resources
- Pricing
- About
- Call‑to‑action button: **Book a Demo**

Footer navigation:

- Platform
- AI Security Tools
- Consulting
- Training
- Resources (Blog, Guides, Webinars)
- Company (About, Careers, Contact)
- Legal (Privacy, Terms, DPA/DPDPA Notice)

This mirrors successful B2B SaaS layouts where core product, pricing, and resources are always 1–2 clicks away.[^2][^1]

### 1.2 Visual design system

- **Color palette**
    - Primary: Deep indigo / navy (\#090E2A or similar) for backgrounds.
    - Accent 1: Electric teal (\#24D1C5) for CTAs, links, highlights.
    - Accent 2: Soft saffron (\#FFC766) for subtle emphasis on badges, icons.
    - Neutrals: Light grey (\#F5F7FB), mid grey (\#A0A4B8), off‑white (\#FFFFFF).
- **Typography**
    - Heading: Modern geometric sans (e.g., Space Grotesk / Poppins style).
    - Body: Clean sans (e.g., Inter / Roboto style).
    - Use large, airy headings (H1 40–52px, H2 28–32px) and 16–18px body text, similar to top SaaS sites.[^3][^4]
- **UI style**
    - Flat, minimal UI with subtle glassmorphism cards on dark gradient backgrounds.
    - Use subtle motion (micro‑animations on hover, scroll‑based fade‑ins) but keep it fast.
    - Rounded card corners (8–12px), soft shadows only on interactive elements.
- **Imagery**
    - Abstract AI/data illustrations: nodes, bridges, shields, flows, compliance icons.
    - No stock “people shaking hands”, focus on system diagrams and dashboards.[^1]
- **Logo usage**
    - Use the **Saola AI Labs** shield‑bridge logo in top‑left on a white or dark nav bar.
    - On dark background: full‑color logo; on light background: navy or monochrome version.(see the generated image above)

    - “Built for Indian SMBs” / “AI Governance, Privacy, Security” / “Fast self‑service onboarding”.


### 2.2 Section: “Why SMBs struggle with AI today”

Two‑column layout: left = icons + problem bullets, right = a brief micro‑case.

Problems (India SMB context):

- “Shadow GenAI” everywhere: teams using AI tools without security or compliance review.
- No clear AI governance policy aligned to NIST or industry frameworks.
- Customer and employee personal data flowing into prompts and training sets.
- No central view of GDPR / DPDPA / ISO 42001 obligations across AI initiatives.
- Limited budgets, no in‑house privacy or security team.

Each bullet has a concise elaboration (1–2 short sentences). This mirrors high‑performing SaaS sites that quickly dramatize the pain.[^2]

    - “NIST AI RMF based governance design.”

Include a **“Compare our capabilities”** link to a section or page with more detailed feature tables.

4. **Demonstrate** – Maintain evidence, dashboards, and reports for management and regulators.

Each step: short text, small icon, “See how it works” modal or link.

## 3. Platform page – “Saola AI Platform”

Route: **/platform**

### 3.1 Above‑the‑fold summary

- H1: “One platform for AI‑era privacy and compliance.”
- Sub: “Operationalize GDPR, DPDPA, ISO 42001 and emerging AI governance obligations—without hiring a large team.”

Two CTAs: **Book a Demo**, **Download Feature Sheet (PDF)**.

### 3.2 Core modules (card grid)

Use a 2×2 grid of feature blocks:

1. **Compliance Workspace**
    - Centralized GDPR / DPDPA / ISO 42001 obligations mapped to AI use cases.
    - Dynamic “readiness” score per regulation.
    - Control library with pre‑mapped tasks, owners, and evidence.
2. **AI Use Case Register**
    - Inventory of all AI and GenAI implementations across teams.
    - Risk rating (low/medium/high) with rationale.
    - Links to DPIA/impact assessments and policies.
3. **Data \& Consent Flows**
    - Data mapping: where personal data is collected, processed, stored, shared.
    - Consent and privacy notices registry.
    - Out‑of‑the‑box templates for SMB websites and apps.
4. **Policy \& Evidence Hub**
    - Store AI, security, and privacy policies in one place.
    - Track training status, attestations, and sign‑offs.
    - Exportable audit reports.

### 3.3 Feature table (for clarity)

| Area | SMB need | Platform feature |
| :-- | :-- | :-- |
| GDPR / DPDPA | Structured readiness view | Guided assessments, gap analysis |
| ISO 42001 | AI \& GenAI governance | Mapped controls and evidence tasks |
| Data privacy | Map data and notices | Data flow builder, templates |
| Reporting | Show compliance to clients | One‑click PDF and CSV exports |

### 3.4 Integrations section

- “Connect with what you already use”
- Logos/labels of typical SMB tools: Google Workspace, Microsoft 365, Slack, Jira, GitHub, common CRM/helpdesk tools (as roadmap if not yet live).
- Short note on using APIs / webhooks to sync use cases and evidence.

## 4. AI Security Tools page

Route: **/ai-security-tools**

### 4.1 Hero

- H1: “Secure your AI and GenAI before attackers or auditors arrive.”
- Sub: “Policy‑driven controls that protect prompts, data, and AI outputs across your stack.”


### 4.2 Tool clusters

1. **Prompt \& Data Guard**
    - Enforce rules on what data can be sent to external LLMs.
    - PII/PHI detection and masking.
    - Policy‑based blocking or redaction.
2. **Model Access Control**
    - Role‑based access to AI tools and APIs.
    - Logging and monitoring of sensitive operations.
    - Alerts for anomalous use patterns.
3. **AI Risk Dashboard**
    - Unified view of AI risks by use case, team, and data type.
    - Integration with the platform’s AI Use Case Register.
    - Exportable risk reports.

Each block has: icon, short explanation, 3 bullets, and a “See example scenario” link.

### 4.3 Example scenario (India SMB)

A short narrative block:

- “Sales team starts using GenAI to draft proposals, including client data.”
- Pain: No restrictions, data leaking into third‑party LLMs, no audit trail.
- With Saola AI:
    - Policies block sensitive fields, mask PII, log all prompts, categorize outputs.
    - Compliance workspace shows this use case and its controls.

## 7. Pricing page

Route: **/pricing**

Goal: Self‑service tiers for platform + option to add consulting and training.[^5][^6][^7]

### 7.1 Pricing model

Three self‑service tiers for the platform, plus “Enterprise / Custom”.

Suggested structure (monthly or annual, with INR and USD toggle):

1. **Starter (for small teams)**
    - Up to X employees / Y AI use cases.
    - Core compliance workspace (GDPR, DPDPA lite).
    - Basic AI Use Case Register.
    - Email support.
    - CTA: **Start Free Trial**.
2. **Growth (for scaling SMBs)**
    - Everything in Starter.
    - ISO 42001 module.
    - AI Security Tools (Prompt \& Data Guard basic).
    - Policy \& Evidence Hub.
    - Email + chat support.
    - CTA: **Start Free Trial**.
3. **Scale (for larger or regulated orgs)**
    - Everything in Growth.
    - Advanced AI Security Tools and integrations.
    - Custom workflows, SSO, audit exports.
    - Priority support, dedicated CSM.
    - CTA: **Talk to Sales**.
4. **Consulting \& Training (Add‑ons)**
    - “From ₹X,000 per day or packaged engagements.”
    - “Request a proposal” CTA linking to a short form.

Below the tiers, include an FAQ:

- “Can I start with Platform only and add consulting later?”
- “Do you offer discounts for annual billing?”
- “What if I’m just experimenting with AI?” etc.
